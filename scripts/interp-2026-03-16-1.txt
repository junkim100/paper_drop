HOST1: So here's something that's been bugging me about the superposition story we've all been telling for the last few years. You know the standard picture, right? Neural networks pack more features than they have dimensions, there's interference between them, and the network has to use nonlinearities like ReLUs to clean up the mess. It's a beautiful, clean theory. Except it might be wrong. Or at least, seriously incomplete.
HOST2: Wait, wrong how? The whole sparse autoencoder movement is built on that theory.
HOST1: Right, and that's what makes this paper so interesting. It's from a team at Imperial College London, led by Lucas Prieto. The title is "From Data Statistics to Feature Geometry: How Correlations Shape Superposition." And the key insight is this: the original superposition theory from Elhage and Olah's 2022 paper assumed features are sparse and uncorrelated. Independent of each other. But real data isn't like that at all.
HOST2: Of course it isn't. If you're reading about December, you're probably also reading about Christmas and New Year and winter. Features co-occur all the time.
HOST1: Exactly. And nobody had really checked what happens to the geometry of superposition when you account for that. So this team built a framework called BOWS, which stands for Bag-of-Words Superposition. It's a controlled setting where you take actual internet text, specifically WikiText-103, and represent each chunk of text as a binary bag of words. Is the word "sun" present? Yes or no. Is "Christmas" present? Yes or no. Ten thousand vocabulary words total.
HOST2: So you get all the real correlation structure of language, like "December" and "Christmas" co-occurring, but in a clean enough format that you know the ground truth features.
HOST1: That's the beauty of it. You have realistic feature correlations but you can still fully analyze what the model is doing because you know every feature. Then they train a simple autoencoder, the same kind Elhage used in the original toy model work, where a linear encoder projects ten thousand features down to fewer dimensions, and a decoder with a ReLU tries to reconstruct them.
HOST2: OK so what happens when features are correlated? What changes about the geometry?
HOST1: This is the big finding. When features co-activate, meaning they tend to show up together in the same text, the model arranges them so that the interference between those features is actually constructive. Not noise to be suppressed, but signal that helps reconstruction.
HOST2: Hold on. So you're saying the interference, which the whole superposition theory treats as the villain, the thing the ReLU is there to clean up, is actually doing useful work?
HOST1: For correlated features, yes. Think of it this way. In the old picture, if "December" and "Christmas" end up with overlapping directions in the embedding space, that's bad. Their signals bleed into each other. The ReLU has to figure out which one is really active and suppress the false signal from the other. But in the new picture, if "December" and "Christmas" always show up together anyway, then the bleed-through from "December" into "Christmas" reconstruction is actually helpful. It's free signal. The model doesn't need to suppress it. It can ride it.
HOST2: So basically the model is surfing the interference instead of fighting it.
HOST1: That's a great way to put it. And the paper formalizes this as what they call "linear superposition." In this regime, you don't even need the ReLU to recover features. A purely linear decoder can do it, as long as the data has enough low-rank structure, meaning the features are correlated enough that most of the variance lives in a small subspace.
HOST2: But you'd still need the ReLU for some features, right? Not everything is correlated.
HOST1: Right, and this is where it gets really interesting. In practice, you get both mechanisms coexisting in the same model. Correlated features use constructive interference, and sparse uncorrelated features still rely on the classic ReLU-based filtering. The paper shows clean examples of this. When you train with twelve synthetic features that have a cyclic correlation pattern, like the months of the year where January is close to February, February to March, and so on, the model at small bottleneck sizes uses purely linear superposition. It projects everything onto a circular structure that matches the correlation pattern. But at larger bottleneck sizes where you have more room, the ReLU model switches to forming antipodal pairs, the classic interference-minimizing geometry.
HOST2: So the model is choosing the strategy that works best given the available dimensions and the correlation structure. That's really elegant. But what does weight decay have to do with it? You mentioned that earlier.
HOST1: This is a key practical finding. Weight decay, which penalizes large weight norms during training, strongly amplifies the constructive interference regime. And the reason is intuitive once you think about it. The classic superposition strategy, where you use the ReLU to suppress interference, needs each feature to have roughly unit-norm weight vectors. So the total weight norm scales with the number of features. But constructive interference can exploit low-rank structure in the covariance matrix, which means you can reconstruct features using a much lower-rank weight matrix. Lower rank means lower weight norm. So weight decay naturally pushes the model toward solutions that exploit correlations.
HOST2: OK but here's the thing I really care about. Does this explain stuff we've actually seen in real language models?
HOST1: That's the payoff. People have found these semantic clusters in real LLMs for years. Groups of related features that cluster together in embedding space instead of spreading out to minimize interference. Months of the year forming circles. Related concepts living near each other. The standard superposition theory couldn't explain why. It predicted features should spread out to minimize dot products, forming polytope-like structures. But this paper shows that if you account for correlations, the theory naturally predicts exactly the semantic clusters and cyclical structures that people have observed. "December" is close to "Christmas" in embedding space not because the model failed to separate them, but because it's exploiting their correlation.
HOST2: So wait, does this have implications for how we train sparse autoencoders? Because if interference is sometimes constructive, then aggressively trying to decompose everything into independent features might actually be throwing away useful structure.
HOST1: The paper doesn't go deep into SAE implications, but I think that's the obvious next question. If correlated features are sharing information through constructive interference, then an SAE that forces everything into independent sparse features might be missing part of the story. You might need to think about feature groups, not just individual features.
HOST2: What are the limitations though? This is still a toy setting, right?
HOST1: It is. BOWS is more realistic than the fully synthetic setup from the original toy models paper because it uses real text statistics. But it's still bag-of-words features in a single-layer autoencoder, not a full transformer. The paper acknowledges this. Bridging from BOWS to actual transformer internals is the big open challenge.
HOST2: Still, as a theory paper, this feels like a pretty significant update to the superposition picture. Four years of everyone building on the "interference is noise" assumption, and it turns out that was only half the story.
HOST1: Exactly. The original Elhage paper gave us the foundation. This one adds the missing floor. When your features are correlated, and they always are in real data, interference isn't just a cost to be minimized. It's a resource to be exploited. And I think that reframe is going to change how people think about feature geometry and maybe even how they design the next generation of interpretability tools.
HOST2: Good stuff. I'm curious to see where this leads, especially once people start testing these predictions on actual transformer representations rather than toy models.
HOST1: Same. The code is already public on GitHub, so I expect we'll see follow-up work pretty quickly.
